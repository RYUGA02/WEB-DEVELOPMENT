JavaScript IIFE (Immediately Invoked Function Expression)

What is an IIFE?


An Immediately Invoked Function Expression (IIFE) is a JavaScript function that runs as soon as it is defined. It helps in encapsulation, avoiding polluting the global scope, and executing code immediately.


Syntax of an IIFE


An IIFE is wrapped inside parentheses and invoked immediately:


(function() { // Code inside this function runs immediately })();


or using arrow function syntax:


(() => { console.log("IIFE executed!"); })();


Why Use an IIFE?


1.Encapsulation (Avoids Global Scope Pollution) Variables declared inside an IIFE cannot be accessed outside, preventing namespace conflicts.


2.Execution Immediately Executes as soon as the script is loaded, useful for initialization tasks.


3.Avoids Variable Hoisting Issues Ensures variables and functions inside do not get hoisted to the global scope.


4.Helps in Module Pattern Useful for creating modules in JavaScript.


Examples of IIFE


1. Basic Example (function() { console.log("Hello, this is an IIFE!"); })();


Output: Hello, this is an IIFE!


Conclusion


  • IIFE is a self-executing function that prevents global scope pollution.


  • It helps in modular programming and data privacy.


  • It is commonly used for initialization, encapsulation, and private variables.
