JavaScript Strict Mode ("use strict")

What is Strict Mode?

Strict Mode in JavaScript is a way to enforce stricter parsing and error handling in your code. It was introduced in ECMAScript 5 (ES5) to help developers write more secure and optimized JavaScript. It prevents the use of potentially problematic syntax and improves performance by catching silent errors that might otherwise go unnoticed.


To enable Strict Mode, you use:
"use strict";
or
'use strict';
This directive must be written at the beginning of a script or a function; otherwise, it will not be applied.

Why Use Strict Mode?

    Strict Mode helps in the following ways:
  • Prevents the use of undeclared variables

  • Throws errors for assigning values to read-only properties

  • Eliminates this coercion in functions

  • Restricts usage of duplicate parameter names

  • Prevents deletion of variables, functions, and certain properties

  • Disallows usage of reserved words for future versions

  • Detects assignment to non-writable properties

  • Disables with statement

  • Catches common coding mistakes
